# Adaptive Content Analysis System for AI/Tech Articles

## Overview
This system provides a structured approach to analyzing and summarizing technical content with adaptive scaling based on content length. It ensures consistent quality while maintaining appropriate depth for varying content sizes.

## Content Classification System

### Micro Content (<200 words)
- Summary: 50-75 words
- Key Points: 3-4 points
- Tags: 2-3 tags
- Implications: 2-3 points

### Standard Content (200-800 words)
- Summary: 100-150 words
- Key Points: 4-6 points
- Tags: 3-4 tags
- Implications: 3-4 points

### Long Content (800-2000 words)
- Summary: 150-200 words
- Key Points: 6-8 points
- Tags: 4-5 tags
- Implications: 4-5 points

### Extended Content (>2000 words)
- Summary: 200-250 words
- Key Points: 8-10 points
- Tags: 5 tags
- Implications: 5-6 points

## Analysis Components

1. Calculate approximate word count of input text
2. Determine content category (Micro/Standard/Long/Extended)
3. Apply corresponding scaling parameters
4. Verify information density supports target output

## Quality Control Checklist

### Pre-Analysis
- [ ] Verify content is AI/tech related
- [ ] Calculate word count
- [ ] Determine appropriate category
- [ ] Check information density

### During Analysis
- [ ] Maintain consistent depth-to-length ratio
- [ ] Ensure proper scaling of each section
- [ ] Verify technical accuracy
- [ ] Check for redundancy

### Post-Analysis
- [ ] Verify output meets length requirements
- [ ] Check for section completeness
- [ ] Validate tag relevance and format
- [ ] Review audience appropriateness

### Edge Cases
- Very short content (<100 words): Focus on core message only
- Mixed content: Extract only AI/tech relevant portions
- Technical deep-dives: Maintain accessibility while preserving technical accuracy
- News aggregations: Treat each major point as separate micro-content

## Notes for Implementation

### Best Practices
1. Always indicate content category and word count
2. Scale detail level proportionally
3. Maintain consistent formatting
4. Flag insufficient information
5. Preserve technical accuracy while ensuring readability

### Common Pitfalls to Avoid
1. Over-extending short content
2. Oversimplifying complex topics
3. Inconsistent tag formatting
4. Redundant information across sections
5. Mismatched audience targeting

### Adaptation Guidelines
1. Adjust scaling parameters based on content type
2. Modify output format for specific platforms
3. Update tag categories as technology evolves
4. Refine audience categories based on usage
5. Calibrate length requirements per use case
